# 🏔️ PicTrip

> 사진 한 장으로 여행이 시작되는 이미지 기반 여행 추천 플랫폼

기존 여행 서비스는 "강릉 바다", "제주 카페"처럼 검색어를 알아야 한다. PicTrip은 장소명을 몰라도 사진 한 장이면 된다. 과거 여행 사진, SNS에서 본 풍경, 드라마 속 장면 — 이미지를 올리면 AI가 분위기를 분석하고 비슷한 여행지를 찾아 일정까지 짜준다.

## 📌 주요 기능

| 기능                  | 설명                                                                                   |
| --------------------- | -------------------------------------------------------------------------------------- |
| 이미지 기반 장소 추천 | 업로드된 사진의 분위기, 풍경, 건축물, 계절감을 AI가 분석해 유사한 여행지 후보 3곳 추천 |
| AI 여행 일정 생성     | 추천 장소 + 여행 조건(기간, 예산, 인원, 교통)을 조합해 방문 가능한 일정 자동 생성      |
| 대화형 재추천         | 결과가 마음에 안 들면 추가 설명을 입력해서 분석/일정 재생성                            |
| 일정 저장·관리        | 생성된 일정 저장, 조회, 수정, 삭제, 공유                                               |
| 후기 커뮤니티         | 원본 이미지와 실제 방문 사진을 함께 기록하는 before/after 후기                         |

## 🚀 설치 및 실행

### 1. 클론

```bash
git clone https://github.com/<!-- 팀 레포 주소 -->.git
cd pictrip
```

### 2. 의존성 설치

```bash
npm install
```

### 3. 환경 변수 설정

```bash
cp .env.example .env
```

`.env` 파일에 아래 값 채우기:

| 변수명                       | 설명                  | 예시                      |
| ---------------------------- | --------------------- | ------------------------- |
| `GEMINI_API_KEY`             | Gemini API 키         | <!-- -->                  |
| `GROQ_API_KEY`               | Groq API 키           | <!-- -->                  |
| `SUPABASE_URL`               | Supabase 프로젝트 URL | `https://xxx.supabase.co` |
| `SUPABASE_KEY`               | Supabase anon key     | `eyJ...`                  |
| <!-- 추가 키 있으면 여기 --> |                       |                           |

### 4. 실행

```bash
npm start
```

`http://localhost:3000` 에서 확인

## 📡 API 엔드포인트

| Method | Endpoint                 | 설명                                                           |
| ------ | ------------------------ | -------------------------------------------------------------- |
| POST   | `/api/auth/signup`       | 회원가입 (이메일/소셜)                                         |
| POST   | `/api/auth/login`        | 로그인 (이메일/소셜)                                           |
| POST   | `/api/auth/logout`       | 로그아웃                                                       |
| POST   | `/api/analyze`           | 이미지 + 보조 설명 + 메타데이터 → AI 분석 + 장소 후보 3개 추천 |
| POST   | `/api/analyze/retry`     | 추가 설명 기반 재분석                                          |
| POST   | `/api/itinerary`         | 선택한 장소 + 여행 조건 → 일정 생성                            |
| POST   | `/api/itinerary/retry`   | 추가 요청 기반 일정 재생성                                     |
| GET    | `/api/trips`             | 저장된 여행 일정 목록 조회                                     |
| POST   | `/api/trips`             | 여행 일정 저장                                                 |
| PUT    | `/api/trips/:id`         | 여행 일정 수정                                                 |
| DELETE | `/api/trips/:id`         | 여행 일정 삭제                                                 |
| GET    | `/api/reviews`           | 후기 목록 조회                                                 |
| POST   | `/api/reviews`           | 후기 작성 (이미지 첨부 가능)                                   |
| GET    | `/api/reviews/search?q=` | 키워드 기반 후기 검색                                          |

## 📅 개발 일정

| 일차      | 핵심 목표                    | 상세 내용                                                                                                                                                                |
| --------- | ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Day 1** | 기획 확정 + 환경 셋업 + 인증 | 요구사항 확정, ERD/DB 스키마 설계, Git 레포 초기화, Express 보일러플레이트, Supabase 연동, 회원가입/로그인/로그아웃 구현 (이메일+소셜), 입력값 검증·로그인 실패 예외처리 |
| **Day 2** | 핵심 기능 구현               | 이미지 업로드·미리보기·파일검증, 메타데이터 추출, AI 이미지 분석 + 장소 3개 추천, 재분석 기능, 장소 선택 + 여행 조건 입력, AI 일정 생성, 필수 조건 누락 안내             |
| **Day 3** | 나머지 기능 + 테스트 + 배포  | 일정 결과 화면, 일정 재생성, 일정 저장/조회/수정/삭제/공유, 후기 CRUD + 검색 + 장소 연결, 외부 API 오류 처리, 기능·예외·통합 테스트, 배포                                |
| **Day 4** | 발표 준비                    | 발표 자료 작성, 시연 데이터 준비 (테스트 계정, 샘플 이미지, 예시 후기), 최종 버그 수정                                                                                   |
| **Day 5** | 발표                         | 시연 + 발표                                                                                                                                                              |

> ⚠️ **Day 3까지 전체 기능 구현 완료가 목표.** Day 4는 발표 준비에만 집중.
